üéØ How It Works

  1. Data Collection: Fetches track metadata, BPM, musical keys, and popularity from multiple APIs
  2. Camelot Conversion: Converts Spotify/GetSongBPM keys to Camelot notation (e.g., 8A, 5B)
  3. Compatibility Scoring: Calculates harmonic, BPM, energy, and popularity scores for each transition
  4. Greedy Sequencing: Uses multi-factor optimization to build the optimal track order
  5. Justification Engine: Explains every transition decision for transparency and learning
